## 🎯 **Objetivo**

Implementar sistema de split de mensagens para dividir textos grandes em parágrafos curtos e enviá-los sequencialmente com delays, simulando digitação humana (não IA).

**User Story:**
> Como usuário do Ensinio Prospector, eu quero que as mensagens sejam enviadas em múltiplas partes curtas (parágrafos) com delays entre elas, para que pareçam mensagens naturais de uma pessoa real, não um blocão de texto gerado por IA.

**Implementação:**
```javascript
async function sendWithSplit(client, phone, message, opts = {}) {
  const parts = splitMessageIntoParts(message);

  for (let i = 0; i < parts.length; i++) {
    const typingDelay = Math.floor(Math.random() * 2000) + 1000; // 1-3s

    console.log(`[${i + 1}/${parts.length}] Enviando parte...`);
    await client.sendText(phone, parts[i], { delay: typingDelay });

    if (i < parts.length - 1) {
      const pauseDelay = Math.floor(Math.random() * 2000) + 2000; // 2-4s
      await sleep(pauseDelay);
    }
  }
}
```

describe('splitMessageIntoParts', () => {

## 💡 **Implementation Notes**

### **Why Split by `\n\n`?**
- Parágrafos são unidades naturais de pensamento
- Fácil de detectar (sem NLP complexo)
- Mensagens já vêm formatadas assim do outreach writer

### **Why Random Delays?**
- Delays fixos são detectáveis (padrão de bot)
- Random 2-4s simula variação humana
- Typing delay adicional simula digitação

### **Why 150 chars max?**
- Leitura confortável no mobile
- Evita parágrafos gigantes mesmo se não tiver `\n\n`
- WhatsApp mostra ~3-4 linhas por mensagem
